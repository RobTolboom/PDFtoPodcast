**Goal:** Fix four structural schema mismatches and three `schema_repair.py` gaps that caused `max_iterations_reached` and false-positive podcast warnings in the pipeline.

**Architecture:** Five targeted changes across four files: (1) widen `key_values` in `common.schema.json` to accept depth-1 objects; (2) make `ContrastResult.effect` optional in two publication-type schemas; (3) add three deterministic repair rules to `schema_repair.py`; (4) fix case-insensitive abbreviation matching in `podcast_logic.py`. Rebundle all five bundled schemas after each JSON schema edit.

**Tech Stack:** Python 3.10+, `jsonschema` (schema validation in tests), `pytest` with `@pytest.mark.unit`, `json-bundler.py` (schema bundling via `make bundle-schemas`).

## Task 4 — `schema_repair.py`: Drop malformed JSON-fragment strings in arrays

**Files:**
- Modify: `src/pipeline/schema_repair.py`
- Test: `tests/unit/test_schema_repair.py`

**Background:** When the LLM truncates its output mid-object, the result is a string like `ontology_terms":[{` in an array that expects objects. The current `_repair_array` tries to look up this string as an ID in the original extraction, fails, and keeps it (the string then fails validation). Strings containing `"`, `{`, `[`, or `:` are always JSON fragments — never valid IDs — and should be dropped.

                "type": "array",

                "type": "object",

        },
    }

## Task 5 — `schema_repair.py`: Flatten depth-2+ objects in `key_values`

**Files:**
- Modify: `src/pipeline/schema_repair.py`
- Test: `tests/unit/test_schema_repair.py`

**Background:** Task 1 widened the schema to allow depth-1 object values in `FigureSummary.key_values`. Depth-2+ nesting (an object whose values are themselves objects) is still invalid. This repair flattens such structures deterministically using `_` as a key separator, turning `{"breakdown": {"a": {"x": 1}}}` into `{"breakdown_a_x": 1}`. Depth-1 objects are left untouched.

def _flatten_key_values_entry(sub_dict: dict[str, Any], prefix: str) -> dict[str, Any]:
    """Recursively flatten a depth-2+ key_values entry into scalar key-value pairs.

    Only called on a single entry value (a dict) that itself contains further dicts.
    Uses '_' as the key separator.

    Args:
        sub_dict: The depth-1 dict value to flatten.
        prefix: The parent key (becomes the leading part of all flattened keys).

    Returns:
        Flat dict, e.g. {"breakdown_a_x": 1, "breakdown_a_y": 2}.
    """
    flat: dict[str, Any] = {}
    for k, v in sub_dict.items():
        full_key = f"{prefix}_{k}"
        if isinstance(v, dict):
            flat.update(_flatten_key_values_entry(v, full_key))
        else:
            flat[full_key] = v
    return flat


def _repair_figures_key_values(figures_summary: list[Any]) -> None:
    """Flatten depth-2+ objects in key_values for each figure. Modifies in-place.

    Iterates each key_values entry. If a value is a depth-1 object whose own
    values are all scalars, it is valid per schema and left alone. Only entries
    whose depth-1 object values contain further dicts (depth ≥ 2) are flattened.
    This preserves valid CONSORT breakdowns (depth-1) while fixing over-nested data.
    """
    for figure in figures_summary:
        if not isinstance(figure, dict):
            continue
        kv = figure.get("key_values")
        if not isinstance(kv, dict):
            continue
        new_kv: dict[str, Any] = {}
        for k, v in kv.items():
            if isinstance(v, dict) and any(isinstance(sv, dict) for sv in v.values()):
                # depth-2+ found: flatten this entry only
                logger.info(
                    "Flattening depth-2+ key_values[%s] in figure %s",
                    k,
                    figure.get("figure_id", "?"),
                )
                new_kv.update(_flatten_key_values_entry(v, k))
            else:
                # scalar or valid depth-1 object: keep as-is
                new_kv[k] = v
        figure["key_values"] = new_kv
